## 0. 핵심 컨셉 (3 줄)

1. 플레이어는 **모바일 수호신** 1체와 함께 움직인다 (위치 기반)
2. 플레이어가 머무는 곳에 **고정 수호탑 13종**을 건설해 영역을 점유한다
3. 적 영역 점령은 **두 가지 경로**로만 가능 — 직접 침투(Solo Infiltration) 또는 공성(Tower Siege)

## 1. 두 가지 정복 경로 (★ 핵심 차별화)

게임의 모든 PvP는 둘 중 하나로 분류된다.

### 경로 A — 직접 침투 (Solo Infiltration)
> "혼자 들어가서 타워와 1:1로 붙어 그 자리를 빼앗는다"

| 단계 | 행동 | 결과 |
|------|------|------|
| 1 | 적 영역 진입 (50m 이내) | 영역 내 타워 자동 발사 시작 (`processTowerDamage`) |
| 2 | 플레이어가 타워 클릭 → "도전" | **본체 ATK vs 타워 DEF/HP** 1:1 전투 |
| 3-A | 타워 1개 격파 | 해당 타워 삭제, **그 자리에 즉시 자기 타워 1개 무료 건설** (영역 소유자는 그대로) |
| 3-B | 같은 영역 모든 타워 격파 | 영역 소유자에게 6h grace 푸시 → 미재건 시 영역 자동 점령 |
| 패배 | 본체 HP 차감, 5분 쿨다운 | 영역 24h vulnerable 진입 (사망 시) |

**특징**
- **빠르고 위험** — 단독 진입, 본체 HP 소모, 즉시 결과
- **보상** — 격파 즉시 storage 약탈, 자리 한 칸 무료 건설권
- **AR 보너스** — 실제 위치 50m 내에서 공격 시 ATK ×1.20
- **전제** — 본체 HP/ATK 충분, 5분 쿨다운 풀린 상태

### 경로 B — 공성전 (Tower Siege)
> "옆에 타워를 박고 멀리서 두들겨 깬다"

| 단계 | 행동 | 결과 |
|------|------|------|
| 1 | 적 영역 인근(사거리 안)에 자기 영역 확장 + 타워 건설 | 사거리 닿는 타워끼리 자동 교전 |
| 2 | 5분 tick마다 자동 발사 (`processTowerSiege`) | 양방향 데미지, 본체 무관 |
| 3 | 적 타워 모두 HP 0 | `siege_breached_at` 설정, 6h grace |
| 4 | grace 내 재건 0 | `siege_last_attacker`에게 영역 자동 점령 (+150 XP) |

**특징**
- **느리고 안전** — 본체 노출 없이 비동기, 오프라인에서도 진행
- **사거리 메타** — 발리스타(150m)·SF(130m)가 핵심 어택커, 자연·크리스탈은 후방
- **자원 부담** — 인접 영역 확장 + 다수 타워 건설 비용
- **카운터** — 방어자가 grace 6시간 안에 타워 1개라도 재건하면 점령 무효

### 경로 비교 매트릭스

| 차원 | 직접 침투 (A) | 공성전 (B) |
|------|---------------|------------|
| 주체 | 본체 수호신 | 자기 타워 |
| 시간 | 즉시 (분) | 비동기 (시간~일) |
| 위험 | 본체 HP / 사망 | 자기 타워 손실 |
| 자원 | AR 이동 | 영역+타워 건설비 |
| 보상 | 빈 슬롯 + 약탈 | 영역 통째 |
| 적합 상황 | 가까운 약한 타워 1~2개 | 멀고 두꺼운 영역 |
| 멀티 동시성 | 1명만 가능 | 동맹·길드 합류 가능 |

> **설계 원칙:** 둘은 **상호 대체가 아니라 보완**. 같은 영역도 상황에 따라 어느 쪽이 효율적인지 다르다. 게임의 결정 지점은 "어느 경로를 택할지"가 되어야 한다.

## 2. 유닛 구조

### 2.1 모바일 수호신 (본체) — **1차 출시는 단일 타입**

플레이어 1명 = 본체 1체. 위치 = 플레이어 GPS 좌표.

**기본 스탯 (Lv1)**
- ATK 10 / DEF 10 / HP 100 / ABS 10% / SPD 10 / RNG 10

**역할**
- 직접 침투(A)의 유일한 공격자
- 영역 확장·타워 건설 트리거 (현재 위치 기준)
- 사망 시(HP 0): HP 50 복구, 모든 영역 24h vulnerable, 5분 쿨다운, `veteran→beginner` 강등

**스탯 캡** ATK 500 / DEF 500 / HP 2000 / ABS 80%

**[추후 — 수호신 확장]**
- 타입 4종 (animal / robot / aircraft / production)
- 상성: animal > aircraft > robot > animal (×1.15 / ×0.87)
- 타입별 궁극기 4종
- AR 3D 본체 표시

> **결정:** 1차 출시는 단일 타입. 다양성은 13종 타워가 제공. 본체 타입 분화는 메타 안정 후.

### 2.2 고정 수호탑 (13종 × 5티어) — **현재 핵심 콘텐츠**

영역당 **최대 3개**. 클래스 × 티어(1~5) = 65 빌드.

| **CC** | 얼음 | 80 | 3s | 2.0 | 50 | 적 영역 vuln 10s | 60 |
| | 아쿠아 | 90 | 3s | 4.0 | 60 | 적 영역 vuln 5분 | 60 |

#### 레벨 스케일 (Lv1 → Lv5)

```
DMG  ×(1 + 0.45 × (L-1))     L5 = ×2.80
HP   ×(1 + 0.55 × (L-1))     L5 = ×3.20
RNG  ×(1 + 0.10 × (L-1))     L5 = ×1.40
연사  ×max(0.6, 1 - 0.08(L-1))  L5 = ×0.68
비용  ×1.4^(L-1)              L5 = ×3.84
```

#### 빌드 가이드 (권장 조합)

| 빌드 | 조합 | 컨셉 |
|---|---|---|
| 공격 거점 | 발리스타 + SF + 자연 | 장거리 정밀 + 인접 회복 |
| 요새 | 돌격×2 + 자연 | 연사 + 회복 |
| CC 함정 | 아쿠아 + 얼음 + 대포 | 묶고 광역 폭발 |
| 시너지 허브 | 크리스탈 + 자연 + 비전 | 동맹 부스트 / 적 디버프 |

#### Storage (생산 누적)
타워마다 cap=5 (+ 레벨 보너스 +1~3). 영역 파츠 드랍은 타워 있으면 storage 누적, 격파 시 약탈됨.

## 3. 영역 시스템

### 3.1 확장

| 반경 | 비용 | 유지비/h | 방어계수 | 패턴 |
|---|---|---|---|---|
| 50m | 10 | 1 | 1.00 | 거점 (도시) |
| 100m | 25 | 3 | 0.50 | 일반 |
| 200m | 60 | 8 | 0.25 | 외곽 |
| 300m | 120 | 15 | 0.13 | 시골/광역 |
| 500m | 250 | 30 | 0.10 | 거대 |

- **방어 계수:** `min(1.0, 50/radius)` — 큰 영역은 약한 방어 (트레이드오프)
- **겹침 차단:** 두 영역 거리 < (r1+r2) → 거부

### 3.2 유지 / 소멸

```
energy = 0       → warning_at 시작
warning + 12h   → weakened_at (방어력 -50%, [TODO] 효과 적용)
warning + 48h   → 영역 + 모든 타워 자동 삭제
energy 회복     → 경고 자동 해제
```

### 3.3 취약 (Vulnerable)

`vulnerable_until` 활성 시 — 방어 ×0.7, 공격 ×1.2.
**트리거**
- 본체 사망 → 모든 영역 24h
- 아쿠아 타워 피격 → 5분
- 얼음 → 10초
- 비전 → 1분

### 3.4 진형 (바둑식)

**Atari (단수)** — 적 영역이 ≥3개 우호 영역으로 둘러싸임
- 매 tick +5 데미지 누적
- 24h 후 첫 공격자에게 자동 점령 (+100 XP, 어시스트 +30)

**Eye (눈)** — 같은 사용자 영역 ≥3개로 둘러싸이고 적 인접 0
- `in_eye_zone=true` (보호 표시)

**시너지** — 우호 연결 1개당 +5%, 캡 +30%
**분산 패널티** — 고정수호탑 N개 → ×1/√N (레벨 freeSlot으로 면제)

## 5. 전투 공식

### 본체 vs 타워 (경로 A) / 본체 vs 본체

```
atkPower = ATK × rand(0.8~1.2)
        × (arMode? ×1.20 : 1)
        × (ult? ×1.5 : 1)
        × typeAdvantage(1.15 / 1.0 / 0.87)
        × vuln.atk(×1.2 if vulnerable)

defPower = (DEF + Σ타워DEF + Σ동맹DEF) × defenseCoeff(radius)
        × rand(0.8~1.2)
        × vuln.def(×0.7 if vulnerable)

승자 = atkPower > defPower
```

### 타워 vs 타워 (경로 B)

```
사거리 안 + 쿨다운(fireRateMs) 경과 → DMG 발사
HP 0 → destroyed_at, 영역 모든 타워 0 → siege_breached_at + 6h grace
grace 만료 + 타워 0 → siege_last_attacker가 영역 점령
```

### 결과 처리

- **승리** — ABS%만큼 ATK/DEF/HP 흡수, 영역 점령 + 모든 타워 삭제, 에너지 +10/-10, battle_wins +1, XP +50
- **사망 (HP≤0)** — HP 50 복구, 모든 영역 24h vulnerable, veteran→beginner 강등
- **쿨다운** — 5분 (`last_battle_at`)

### 보호 / 차단

- **레이어** veteran(승5+영역3) → beginner 공격 차단
- **방어막** `shield_until` 활성 시 공격 거부
- **배신 차단** `betrayal_blocked_until` 24h

## 10. AR 모드 — 두 트랙

### 10.1 Unity AR (앱 안)
- 실제 위치에 자기 타워 3D 모델 표시 (Piloto Studio 13×5=65 프리팹)
- 본체 수호신 3D는 추후 (수호신 시스템 확장 시)
- AR 활성 + 직접 침투(A) → ATK ×1.20
- 시각 전용 + Unity ↔ Web 양방향 메시지 (PLAYER_ENCOUNTER, FIXED_GUARDIAN_ATTACK)
- **AR 타워 배치 (신 API):** ARFixedGuardianPlacer가 `/api/towers/place` 호출.
  - 흐름: AR 진입 → "Place" 버튼 → 영역 선택 → 13종 클래스 picker + 1-5 티어 → 바닥 평면 탭 → 즉시 3D 스폰
  - 발판(slot grant) 모드: `StartFootholdPlacement(grant)` 진입점 — 영역 선택 스킵, Lv1 고정, 무료
  - UI 미설정 시 fallback: generic Lv1 자동 (테스트 가능)

### 10.2 야전 모드 (Field Mode, 웹)
> Unity가 없는 웹 브라우저용 대체 — "AR 모드" 버튼 누르면 자동 활성화

- **카메라 배경:** `getUserMedia({ facingMode: 'environment' })` 후면 카메라
- **컴패스 화살표:** 디바이스 방향(`deviceorientation`) 기반, 가장 가까운 위협 가리킴
- **HUD:**
  - 가장 가까운 위협 카드 (타워/플레이어 + 거리 + 큰 공격 버튼)
  - 위협 리스트 (5개)
  - 발판(grant) 리스트 + 거리 + 잔여시간 → 탭하면 AR 내부에서 즉시 발판 모달
  - 액션 바: 영역 확장 / 타워 배치 / 카메라 토글
- **AR 내 타워 배치:**
  - "🏰 타워 배치" 탭 → 영역 1개면 즉시 모달, 여러개면 picker (반경/거리/슬롯 표시)
  - TowerPlacementModal이 AR 위에 z-index 6000으로 떠서 카메라 배경 유지
  - 13종 + 4 빌드 프리셋 + 5 티어 모두 AR 안에서 사용 가능
- **보너스:** AR 활성 동안 ATK ×1.20 (Unity와 동일)
- **카메라 권한 거부 시:** 검은 그라디언트 배경 + HUD만 표시 (기능 작동)

> WebXR 진짜 AR은 미도입 (iOS Safari 부재 + 자산 재변환 비용). 야전 모드는 "카메라 배경 + 위치 기반 HUD"로 대체.
